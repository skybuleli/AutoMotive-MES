3数据模型 PostgreSQL 17 · Ulid 主键 · 12 表

🗄 核心 DDL

1. production_orders — 生产工单 (聚合根)

CREATE TABLE production_orders ( id UUID PRIMARY KEY, -- Ulid → Guid order_number VARCHAR(32) NOT NULL UNIQUE, -- WO-20260701-0001 product_code VARCHAR(32) NOT NULL, product_name VARCHAR(128) NOT NULL, quantity INTEGER NOT NULL, completed_qty INTEGER NOT NULL DEFAULT 0, defect_qty INTEGER NOT NULL DEFAULT 0, status VARCHAR(16) NOT NULL DEFAULT 'Created', priority SMALLINT NOT NULL DEFAULT 1, routing_id UUID REFERENCES routings(id), equipment_id UUID REFERENCES equipments(id), created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), started_at TIMESTAMPTZ, completed_at TIMESTAMPTZ ); CREATE INDEX idx_orders_status ON production_orders(status); CREATE INDEX idx_orders_created ON production_orders(created_at);

2. traceability_links — 追溯链 (4 级双向)

CREATE TABLE traceability_links ( id UUID PRIMARY KEY, vin_or_serial VARCHAR(64) NOT NULL, -- VIN码 或 成品序列号 order_id UUID NOT NULL REFERENCES production_orders(id), component_batch VARCHAR(64) NOT NULL, -- 关键零部件批次 material_batch VARCHAR(64) NOT NULL, -- 原材料批次 link_level SMALLINT NOT NULL DEFAULT 4, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); CREATE INDEX idx_trace_vin ON traceability_links(vin_or_serial); CREATE INDEX idx_trace_component ON traceability_links(component_batch); CREATE INDEX idx_trace_material ON traceability_links(material_batch);

3. quality_records — 质检记录

CREATE TABLE quality_records ( id UUID PRIMARY KEY, order_id UUID NOT NULL REFERENCES production_orders(id), inspection_type VARCHAR(8) NOT NULL, -- IQC|IPQC|FQC check_point VARCHAR(64) NOT NULL, characteristic VARCHAR(64) NOT NULL, measured_value DECIMAL(12,4) NOT NULL, lower_limit DECIMAL(12,4), upper_limit DECIMAL(12,4), is_passed BOOLEAN NOT NULL, inspector_id UUID NOT NULL REFERENCES users(id), checked_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); CREATE INDEX idx_quality_order ON quality_records(order_id); CREATE INDEX idx_quality_checked ON quality_records(checked_at);

🔑 Ulid 主键说明
所有实体使用 Ulid.NewUlid() 生成可排序 UUID，通过 UlidToGuidConverter 存入 PostgreSQL uuid 列。B+Tree 索引性能比随机 UUID 提升 30~50%，比自增 ID 更适合分布式环境（多工厂数据交换无冲突）。

📊 Effect 策略矩阵

阶段Effect ID策略幂等保护崩溃恢复行为
安全联锁检查—Effect 之外不适用重放时重新评估物理安全
设备就绪确认EnsureReady-{seq}AtMostOnce重放复用上次结果不重复查询 PLC
下发加工参数WriteParam-{seq}-{tag}AtLeastOnce先读后写 (read-before-write)重试直到 PLC 确认
启动加工StartOperation-{seq}AtLeastOncePLC 启动命令幂等重试直到设备启动
过程监控Monitor-{seq}AtMostOnce重放使用上次监控结果不重复轮询
完工确认CompleteOperation-{seq}AtLeastOnceDB 写入幂等 (by PK)重试直到 DB 确认
物料绑定BindMaterial-{batchNo}AtLeastOnce唯一约束 (orderId+batchNo)不重复绑定
质检记录RecordInspection-{id}AtLeastOnce记录 ID 去重不重复记录
⚠️ 核心原则
Effect 内禁止读 Redis/外部缓存；AtLeastOnce 必须幂等保护；安全联锁永远在 Effect 外实时判断。
