⚠️ 为什么 ESP 必须 100% 追溯？
ESP 是汽车安全件（ISO 26262 ASIL-D 最高等级）。一旦发生召回，博世必须在 24 小时内从 VIN 码追溯到 ECU S/N → HCU S/N → 电机 S/N → 电磁阀批次 → 阀体原材料批次。追溯失败 = 无法界定召回范围 = 整车厂将要求全批次召回，损失可达数亿欧元。这是 MES 系统存在的第一性原理。

4M01 生产工单管理 P0 核心
📋 模块概述
管理 ESP 总成从 ERP 下达到成品入库的完整生产工单生命周期。与博世 SAP 系统通过 RFC/BAPI 接口双向同步工单状态。

工单生命周期 9 个步骤

  1. 步骤 1 · 工单创建与校验 — SAP 通过 Webhook 推送工单 → MES 接收并校验产品编码（ESP-9.0/ESP-9.1）、BOM 版本、工艺路线版本 → 验证通过后创建本地工单，分配 Ulid 主键 + 工单号 WO-YYYYMMDD-NNNN → 如 BOM 版本不符则拒单并回写 SAP 异常
  2. 步骤 2 · 物料齐套检查 — 根据 BOM 展开物料清单 → 查询线边库存 + ERP 库存 → 检查 ECU S/N 批次、HCU 阀体批次、电机批次是否足够 500 件 → 缺料则触发 JIT 看板拉动 → 齐套确认通过后工单状态变为 Released
  3. 步骤 3 · 首件检验 — 每班次/换型后强制首件检验 → 班组长扫码选择首件模板 → 按控制计划逐项检验并录入结果 → 系统自动判定合格/不合格 → 不合格则锁定工单，触发 CAR
  4. 步骤 4 · 工单开工 — 班组长在工位终端扫码工单 → 系统分配设备 → 触发 Cleipnir Saga 启动 31 道工序序列 → 状态变为 InProgress
  5. 步骤 5 · 工序执行监控 — Saga 按工艺路线依次执行每道工序 → 每道工序完成时自动记录：操作员工号、设备号、开始/结束时间、过程参数 → 异常时自动触发 Andon
  6. 步骤 6 · 过程巡检 (IPQC) — 按控制计划频率（如每 50 件）自动弹出巡检任务 → 检验员扫码录入实测值 → SPC 控制图实时更新 → 超出控制限自动报警
  7. 步骤 7 · 物料批次绑定 — 每件产品经过关键工位时扫码绑定：ECU S/N → HCU S/N → 电机 S/N → 电磁阀批次 → 阀体批次 → 全部写入追溯链接表
  8. 步骤 8 · 完工确认 — 全部 31 道工序完成 → 系统自动统计合格数/不良数 → 质量工程师审核放行 → 状态变为 Completed
  9. 步骤 9 · 成品入库 — 打印唯一追溯标签（含二维码，编码规则：供应商代码+产品代码+年月日+序列号+校验位）→ SAP 同步完工数量 → 状态变为 Closed

5M02 物料管理 (JIT/JIS) P0 核心
📦 模块概述
管理 ESP 总成所需 87 种物料的批次追溯、线边库存和 JIT 看板拉动。与博世 SAP WM 模块通过 IDoc 接口同步库存数据。

物料管理 7 个核心步骤

  1. 步骤 1 · BOM 版本管理 — 从 SAP/PLM 同步 ESP-9.0 BOM（87种物料，4层结构）→ 本地缓存到 MasterMemory → 工单创建时校验 BOM 版本 → 版本不符则拒单
  2. 步骤 2 · 来料扫码入库 — 仓库员用扫码枪扫描供应商来料标签 → 系统自动解析 GS1-128 条码（含物料编码+批次号+数量+生产日期）→ 校验是否在合格供应商名录内 → 写入 material_bindings 表
  3. 步骤 3 · 线边库存实时监控 — 每个工位配置电子看板 → 显示当前物料库存量、安全库存、最大库存 → 低于安全库存时黄色预警 → 低于最低库存时红色报警 + 自动触发叫料
  4. 步骤 4 · JIT 看板拉动 — 工位扫描空料箱 → 生成电子看板信号 → 推送至仓库 PDA → 仓库员按看板备料 → 送达工位后扫码确认 → 全流程时间戳记录
  5. 步骤 5 · 投料批次绑定 — 操作员在投料前扫描物料批次码 → MES 自动绑定：物料编码+批次号 → 工单号 → 当前产品 S/N → 写入追溯链
  6. 步骤 6 · 物料防错 (Poka-Yoke) — 关键物料（ECU芯片、电磁阀）启用防错校验 → 扫描物料码后与 BOM 比对 → 物料错误时锁定设备并声光报警 → 必须质量工程师解锁
  7. 步骤 7 · 物料消耗反冲 — 工单完工后自动反冲物料消耗 → 按 BOM 标准用量扣减线边库存 → 差异 > 2% 时生成异常报告 → 同步 SAP 物料移动凭证
6M03 质量管理 (SPC + 全检) P0 核心
✅ 模块概述
覆盖 IQC（来料）→ IPQC（过程）→ FQC（成品）三级检验 + SPC 统计过程控制。ESP 总成须 100% 通过液压功能测试（安全件强制要求）。

质量管理 8 个核心步骤

  1. 步骤 1 · 来料检验 (IQC) — 关键物料（ECU芯片、电磁阀、压力传感器）按 AQL 抽样方案检验 → 检验员扫码录入：外观/尺寸/电性能 → 系统自动判定 → 不合格则锁定批次并触发 SQE 8D
  2. 步骤 2 · 首件检验 — 每班次/换型/设备维修后强制首件 → 按控制计划逐项检验（ESP 共 23 个检验特性）→ 全部合格方可批量生产 → 不合格则锁定工单
  3. 步骤 3 · 过程巡检 (IPQC) — 按控制计划频率：螺栓拧紧每 50 件抽 5 件 → 液压压力每 100 件抽 3 件 → 系统自动弹出巡检任务 → 超时未完成升级报警
  4. 步骤 4 · SPC 实时控制图 — 6 个关键特性实时 X̄-R 控制图：M6螺栓扭矩、M8螺栓扭矩、液压建压时间、保压压力、泄漏率、CAN通信延迟 → R3 响应式管道：每 5 样本自动计算 Cpk → 超限自动报警
  5. 步骤 5 · 100% 在线测试 — 液压功能测试 100% 全检（ESP 安全件强制）→ 12 路电磁阀逐一动作测试 → 建压/保压/泄压循环 → 泄漏率 ≤ 0.5 CC/hour → 任一不合格则设备自动锁止
  6. 步骤 6 · 不合格品处置 — 不合格品自动移入隔离区 → 质量工程师评审：返工/让步/报废 → 返工品经重新检验合格后回线 → 全流程可追溯（原始不合格+返工+复检）
  7. 步骤 7 · 8D/CAR 闭环 — 重复性问题自动触发 8D 报告流程 → D1~D8 每个步骤责任人和截止日期 → 超时自动升级 → 永久纠正措施验证后关闭
  8. 步骤 8 · 质量报表自动生成 — 日报：一次合格率+不良品分布 → 周报：Cpk 趋势+控制图+异常总结 → 月报：PPM+供应商质量排名+质量成本 → PDF 自动邮件推送

// R3 响应式 SPC 管道 var spcStream = _plcStream .Where(s => s.CheckPoint == "M8-Torque") .Buffer(5) // 每5个样本一组 .Select(samples => { var xbar = samples.Average(s => s.Value); var r = samples.Max(s => s.Value) - samples.Min(s => s.Value); return new SpcResult(xbar, r); }) .Subscribe(async result => { // X̄-R 控制限计算（基于历史120个样本） // UCL_Xbar = X̄̄ + A2 × R̄ = 22.5 + 0.577 × 0.8 = 22.96 Nm // LCL_Xbar = X̄̄ - A2 × R̄ = 22.5 - 0.577 × 0.8 = 22.04 Nm // Cpk = min((USL-X̄̄)/3σ, (X̄̄-LSL)/3σ) = min(0.67, 2.03) = 1.67 if (result.XBar > 22.96 || result.XBar < 22.04) { await _andon.Trigger("M8扭矩超控制限", "critical"); } await _push.PublishSpcSample(result); });

OEE 实时计算管道（R3响应式）

  1. 数据采集：OpcUaPlcClient 每 500ms 读取设备状态寄存器 → 发布 PlcDataChanged 消息到 MessagePipe
  2. OEE计算：OeeReactivePipeline 订阅 PlcDataChanged → Sample(5s) 采样 → 计算可用率（实际运行时间/计划运行时间）、性能率（理想节拍×合格数/实际运行时间）、良品率（合格数/总生产数）
  3. 实时推送：计算结果通过 MessagePipe → DashboardPushService → SignalR OeeUpdated 事件 → 管理后台实时更新
  4. 预防性维护：基于运行时间/次数自动触发维护工单（拧紧机每 10 万次标定、液压台每月密封件更换）
  5. 备件关联：维护工单自动关联备件清单 → 检查备件库存 → 不足则生成采购申请
